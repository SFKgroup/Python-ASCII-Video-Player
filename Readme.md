# Python ASCII Video Player

[中文](./readme.md) [English](./readme_en.md)

鉴于Python低下的运行效率和`print()`控制台输出的局限，人们往往认为Python无法制作动态的ASCII Art图像，这是过分悲观的。事实上Python速率的潜力原比想象的大得多，使用`sys.stdout`代替`print`就能使得控制台输出速率达到可观的地步，配合上JIT技术优化循环速率可以能够轻松地在传统控制台字符大小的情况下使得帧率达到60FPS甚至更多，即使缩小字号(增大一行显示的字符数)也能够在大部分情况下保持24FPS。

### 运行参数

```bash
python main.py -s /your/video/dir
```

`-s`, `--source` 视频文件路径，默认为 **"0"** 表示调用电脑的0号摄像头，支持网络地址
`--width` 画面宽度(单位:字符)，默认为 **0** 表示充满命令行窗口
`--height` 画面高度(单位:字符)，默认为 **0** 表示充满命令行窗口
`--style` 画面类型，默认为 **"char_bw"** 各类型含义如下：

 - `"char_bw"`   	 : 使用黑白两种颜色，通过字符表现明度，帧率最高
 - `"char_color"`  : 使用多种颜色，通过字符类型表示明度，字符颜色表示颜色，帧率较低
 - `"color"`             : 使用多种颜色，通过背景色表示像素颜色，帧率中等
 - `"bw"`     		      : 使用灰度色，通过背景色表示像素颜色，帧率中等

`--charset` 使用的字符集，默认为 **" .:-!=+*#%@"** ，仅在`"char_bw"`和`"char_color"`模式下生效
